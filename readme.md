# 東京理科大学マインクラフトサークル（仮） サーバー構築・運営企画書

## 1. 概要

- **プロジェクト名：** 東京理科大学マインクラフトサークル（仮）サーバー構築・運営プロジェクト

    

- **目的：**

    

    1. **デジタル部室の創出：** 日々の学業・研究の息抜きに、部員がいつでもふらっと集まってダラダラ雑談・冒険できる居心地の良い空間を提供する。

        

    2. **技術的挑戦とイベント展開：** 学祭（理大祭）でのキャンパス再現プロジェクトなどを通設し、学科を越えた共同開発の場とする。

        

- **基本方針：** **「スモールスタート」かつ「省管理（ローコスト＆オートメーション）」**。過疎化リスクや管理者のリソース逼迫を防ぐため、最初から巨大なインフラは構えず、人が集まる仕組みと自動化の仕組みを最初から組み込む。

## 2. 技術スタック（初期フェーズ）

初期フェーズは既存のアセット（M1 Mac）をフル活用し、Dockerによるコンテナ環境で安全かつクリーンに構築します。

|**レイヤー**|**採用技術**|**選定理由**|

|---|---|---|

|**ホストOS**|macOS (M1 / Apple Silicon)|圧倒的な単体スペックと省電力性能。初期費用ゼロ。|

|**プラットフォーム**|**Docker / Docker Compose**|環境を汚さず、将来のVPS移行を数分で完了させるため。|

|**サーバーコア**|**Paper (最新安定版)**|超軽量。Java版・統合版のクロスプレイや各種プラグインの基盤。|

|**公開ネットワーク**|**playit.gg**|ルーターのポート開放不要。無料のカスタムドメイン運用が可能。|

### ■ 主要導入プラグイン

- **GeyserMC + Floodgate:** Java版サーバーにスマホやSwitch（統合版）から接続可能にする。

    

- **Multiverse-Core:** 単一のサーバー内に独立した複数の世界（ワールド）を生成・管理する。

    

- **CoreProtect:** 誰が・いつ・どこで・何をしたかのログを全自動記録。荒らし行為をコマンド1発で時間を巻き戻して修復。

    

### ■ Discord連携と権限設計（DiscordSRV + LuckPerms）

Discordをサークルの認証プラットフォームとして中央集権化し、管理作業を自動化します。

#### ① DiscordSRVによる双方向連携

- **チャット同期:** Discordの特定チャンネルとマイクラ内の全体チャットをリアルタイム同期。「誰かが遊んでいる雰囲気」をDiscord側に可視化し、ログインを促す。

    

- **ログイン通知:** 誰がログイン・ログアウトしたか、誰がどんな実績を解除したかをDiscordへ自動通知。

    

#### ② Discord認証によるロール（権限）の完全同期

マイクラの権限管理プラグイン（LuckPerms）とDiscordSRVを連携させ、**Discord上のロールに応じたゲーム内権限の自動割り当て**を行います。これにより、管理者が手動でホワイトリストを登録する手間をゼロにします。

```

[Discordのロール]                    [マイクラ内の権限・称号]

● Admin (管理者)     ---------->   OP権限（全コマンド解放、荒らしログ確認、世界管理）

● User (現役生)      ---------->   通常サバイバル/建築権限、ホワイトリスト自動登録

● OB (卒業生)        ---------->   読み取り専用（観光）権限、または一部制限付きプレイ

```

- **安全性の担保:** Discordサーバー（学内限定など）に参加し、1回認証コマンドを叩いたユーザーだけが自動的にマイクラサーバーにログイン可能になる仕組み。部外者の突発的な荒らしを完全にシャットアウトします。

## 3. 運営・ワールド構成

### ■ ワールド構成（マルチバースによる「シーズン制アーカイブ」）

過疎リスクへの対策と、4年周期で人が入れ替わる大学サークル特有の「土地の過密化・OB建築の残留問題」をクリアするため、世界を完全に分離・管理します。

```

[ 共通ロビーワールド ]

   ├──> [ シーズン2026 生活ワールド ]（今年のメイン・現役生が自由に開拓）

   ├──> [ 学祭企画ワールド ]（クリエイティブ・キャンパス再現専用エリア）

   ├──> [ 資源採取ワールド ]（数ヶ月ごとに完全リセット・アプデ追従用）

   └──> [ 過去シーズンアーカイブ ]（読み取り専用・OBの建築を文化資産として永久保存）

```

- **新入生の居住区確保:** 1〜2年ごとに新しい「生活ワールド」を横に並列生成するため、新入生は常に未開の綺麗な大地からスタートできます。

    

- **OBの資産保護:** 過去のワールドは「読み取り専用」としてそのまま残すため、OBがいつでも昔の拠点を観光しに帰ってくることができます。

    

### ■ 学祭（理大祭）向けの特別企画

- **内容：** 「マインクラフトで再現する神楽坂/野田/葛飾キャンパス（※所属に準ずる）」

- **運用：** 学祭ワールドを「クリエイティブモード」に設定。Discordで設計図や実際のキャンパス写真、Google Earthのデータを共有しながら、部員全員で寄ってたかって建築する巨大プロジェクトとして進行。学祭当日は、一般来場者がサーバーに観光ログインできるような特設ゲートを設置する。

    

### ■ 資金・コスト計画

- **初期フェーズ（自宅Mac運用）：**

    

    - サーバー維持費（電気代・PC代）：**0円**（既存資産の活用）

        

    - ドメイン代（例: `.club` や `.com`）：**年間 約1,500円〜3,000円**

        

    - 負担方法：発起人の自己負担、または初期メンバーでの割り勘。

        

- **スケールアップフェーズ（VPS移行後）：**

    

    - サーバー維持費：**月額 約1,500円〜3,000円**（4GB〜8GBプラン想定）

        

    - 負担方法：メンバーから「部費」として年間数百円ずつ集金、または学内の公認サークル化による部費申請を目指す。

## 4. 将来の展望（スケールアップ）

サークルが軌道に乗り、常時接続人数が増えてきた段階で以下のステップへシフトします。

### ① クラウド・VPS化へのワンタッチ移行

- Docker環境で完結しているため、ConoHa VPSやXserver VPS、あるいはAWS等の環境へ、`docker-compose.yml` と `data` フォルダを丸ごとコピーするだけで移行が完了します。24時間365日、管理者のPC起動状況に依存しない「真の部室」へと昇華させます。

    

### ② 理科大生ならではの「インフラ工学ハック」

サーバーがスケールした際は、部員内のエンジニア（情報系学生など）の実験場として、以下のような高度な自動化システムを組み込み、技術的なサークルとしての実績を作ります。

- **分散処理と負荷監視:** サーバーの負荷（TPS）やメモリ使用量をGrafana等で可視化するモニターの構築。

    

- **Discord Botによるサーバー遠隔制御:** AWS等のクラウドに移行した場合、ログイン人数が0人の状態が続くと自動でサーバーインスタンスを停止し、Discordで「`/start`」と打つとLambda経由でサーバーが自動起動する「超低コスト運用システム」の自作。
